# 捧月2026决赛 — 选手开发指南

## 一、概述

决赛阶段，你的 Agent 将运行在 Windows 电脑上，通过 ADB 连接 Android 手机，实时操作手机，完成指定任务。只需要编写一个 `agent.py` 文件，继承框架提供的 `BaseAgent` 基类，其余由框架自动完成。

### 对比初赛的关键变化

| 维度 | 初赛 | 决赛 |
| :--- | :--- | :--- |
| 输入 | 数据集截图文件 | 手机实时 ADB 截图 |
| 动作空间 | 5 种（CLICK/SCROLL/TYPE/OPEN/COMPLETE） | 11 种（新增 6 种） |
| 验证方式 | ref.json 程序自动判分 | 评委查看执行过程后人工打分 |
| agent_base | 签名验证/防篡改/双模式 | 大幅简化，仅通过 `VLM_API_KEY` 环境变量配置 |
| 坐标 | 归一化 [0,1000] | 同初赛，归一化 [0,1000] |
| 迁移成本 | — | 零改动复用初赛核心逻辑 |

## 四、核心接口：BaseAgent

### 4.1 类结构

你需要继承 `agent_base.py` 中的 `BaseAgent`，并实现 `act()` 方法。`Agent` 类的基本结构如下：

```python
from agent_base import BaseAgent, AgentInput, AgentOutput, UsageInfo
from agent_base import (
    ACTION_CLICK, ACTION_SCROLL, ACTION_TYPE, ACTION_OPEN,
    ACTION_COMPLETE,
    ACTION_LONG_PRESS, ACTION_DOUBLE_CLICK, ACTION_DRAG,
    ACTION_BACK, ACTION_HOME, ACTION_WAIT,
    VALID_ACTIONS
)

class Agent(BaseAgent):
    def __init__(self, config=None):
        super().__init__(config)
        # 你的初始化逻辑（如初始化消息历史）

    def reset(self):
        """每个测试用例开始前自动调用，重置状态"""
        self.message_history = []

    def act(self, input_data: AgentInput) -> AgentOutput:
        """核心方法：根据输入生成动作"""
        # 1. 构造 messages（调用 generate_messages 或自定义）
        messages = self.generate_messages(input_data)

        # 2. 调用 VLM API（必须使用 self._call_api()）
        response = self._call_api(messages, temperature=0, top_p=0.7)

        # 3. 提取 token 用量（可选）
        usage = self.extract_usage_info(response)

        # 4. 解析模型输出
        content = response.choices[0].message.content
        action, params = self._parse_action_only(content) # 需自行实现解析方法

        # 5. 返回 AgentOutput
        return AgentOutput(
            action=action,
            parameters=params,
            raw_output=content,
            usage=usage
        )
```

### 4.2 AgentInput（输入）

`AgentInput` 是 `act` 方法的输入参数，定义了 Agent 接收的信息：

```python
@dataclass
class AgentInput:
    instruction: str               # 用户指令（如 "去京东买手机"）
    current_image: Image.Image     # 当前手机截图 (PIL Image)
    step_count: int                # 当前是第几步（从 1 开始）
    history_messages: List[Dict]   # 历史消息列表
    history_actions: List[Dict]    # 历史动作列表
    extra: Dict                    # 扩展参数
```

### 4.3 AgentOutput（输出）

`AgentOutput` 是 `act` 方法的返回值，定义了 Agent 执行的动作：

```python
@dataclass
class AgentOutput:
    action: str                    # 动作常量（如 ACTION_CLICK）
    parameters: Dict[str, Any]     # 动作参数（标准格式）
    raw_output: str                # 模型原始输出（用于调试/记录）
    usage: Optional[UsageInfo]     # Token 使用信息（可选）
```

### 4.4 工具方法

基类提供了以下可直接使用的方法：

| 方法 | 说明 |
| :--- | :--- |
| `self._call_api(messages, **kwargs)` | 调用 VLM API，自动使用统一配置 |
| `self._encode_image(image)` | 将 PIL Image 编码为 base64（自动压缩） |
| `self.generate_messages(input_data)` | 生成 OpenAI 格式的消息列表（可重写） |
| `self.build_system_prompt(instruction)` | 构建 system prompt（可重写） |
| `self.extract_usage_info(response)` | 从 API 响应提取 token 信息 |
| `self.reset()` | 重置状态（可重写） |

**API 配置（已在 `agent_base.py` 中固定，无需关心）：**
- `API_URL = "https://ark.cn-beijing.volces.com/api/v3"`
- `MODEL_ID = "ep-20260415212058-bx48w" # 豆包 Vision 模型`
- `API_KEY = 从环境变量 VLM_API_KEY 读取`

## 五、11 种标准动作

### 5.1 初赛保留（5 种）

| 动作 | 常量 | 参数 | 示例 |
| :--- | :--- | :--- | :--- |
| 点击 | ACTION_CLICK / "CLICK" | `{"point": [x, y]}` | `{"point": [500, 300]}` |
| 滑动 | ACTION_SCROLL / "SCROLL" | `{"start_point": [x, y], "end_point": [x, y]}` | `{"start_point": [500, 700], "end_point": [500, 300]}` |
| 输入 | ACTION_TYPE / "TYPE" | `{"text": "内容"}` | `{"text": "搜索"}` |
| 打开应用 | ACTION_OPEN / "OPEN" | `{"app_name": "应用名"}` | `{"app_name": "京东"}` |
| 完成 | ACTION_COMPLETE / "COMPLETE" | `{}` | `{}` |

### 5.2 决赛新增（6 种）

| 动作 | 常量 | 参数 | 说明 |
| :--- | :--- | :--- | :--- |
| 长按 | ACTION_LONG_PRESS / "LONG_PRESS" | `{"point": [x, y], "duration": 1000}` | duration 单位毫秒，默认 1000 |
| 双击 | ACTION_DOUBLE_CLICK / "DOUBLE_CLICK" | `{"point": [x, y]}` | 快速连续点击两次 |
| 拖拽 | ACTION_DRAG / "DRAG" | `{"start_point": [x, y], "end_point": [x, y]}` | 类似滑动但速度较慢 |
| 返回 | ACTION_BACK / "BACK" | `{}` | 按系统返回键 |
| 回桌面 | ACTION_HOME / "HOME" | `{}` | 按系统 Home 键 |
| 等待 | ACTION_WAIT / "WAIT" | `{"seconds": 2}` | 等待指定秒数后再继续 |

## 六、坐标规范

所有坐标必须使用归一化坐标，范围 `[0, 1000]`。
- `[0, 0]` = 屏幕左上角
- `[1000, 1000]` = 屏幕右下角
- `[500, 500]` = 屏幕中心

框架内部会自动将归一化坐标转换为实际像素坐标（根据 ADB 获取的屏幕分辨率）。

## 七、从初赛代码迁移

**零改动复用**

如果你初赛的 `agent.py` 中的 `act()` 方法是以下形式，则完全不需要改动即可在决赛框架中运行：

```python
def act(self, input_data: AgentInput) -> AgentOutput:
    messages = self.generate_messages(input_data)
    response = self._call_api(messages, temperature=0, top_p=0.7)
    # ... 解析 ...
```

**需要适配的地方**

1.  **System Prompt 中的动作描述** —— 建议在 `_build_system_prompt()` 中扩展，以包含所有 11 种动作的描述：

    ```python
    def _build_system_prompt(self, instruction: str) -> str:
        return f"""You are a GUI agent...

    ## Action Space
    click(point='<point>x y</point>')
    long_press(point='<point>x y</point>', duration='ms')
    double_click(point='<point>x y</point>')
    type(content='')
    scroll(start_point='<point>x1 y1</point>', end_point='<point>x2 y2</point>')
    drag(start_point='<point>x1 y1</point>', end_point='<point>x2 y2</point>')
    open(app_name='')
    back()
    home()
    wait(seconds='')
    complete(content='xxx')
    """
    ```

2.  **动作解析逻辑** —— 如果初赛用硬编码 `if "click" 匹配`，需要新增 6 种动作的匹配逻辑。
3.  **首张截图是桌面** —— 决赛框架只执行 `press_home()`，不会自动打开任何应用。你的 Agent 需要通过 OPEN 动作自行打开 `tasks.json` 中指定的应用。

**删除/简化的内容**

以下初赛特有的内容在决赛中已不存在，你的代码中无需关注：
-   签名验证 / 防篡改机制 -> 已移除
-   EVAL_MODE 生产模式分支 -> 已移除
-   FORBIDDEN_KWARGS 过滤 -> 已移除
-   ref.json 参考答案 -> 决赛由评委打分
-   CHECKER 自动验证 -> 已移除

### 8.3 快速单步调试

你也可以写一个简单的测试脚本进行单步调试，例如：

```python

from agent_base import AgentInput

agent = Agent()

# 模拟一次调用
from PIL import Image
img = Image.open("test_screenshot.png")

input_data = AgentInput(
    instruction="去淘宝搜索手机支架",
    current_image=img,
    step_count=1

print(f"Params: {output.parameters}")
```

## 九、提交格式

### 9.1 压缩包命名与结构

文件格式：`submission_<姓名>.zip`

> 必须包含姓名，如 `submission_张三.zip`。评委会根据文件名自动提取选手姓名并创建独立输出目录。

压缩包内部结构：

```text
submission_<姓名>.zip
├── agent.py               # ★ 必须：你的 Agent 实现
├── requirements.txt       # 可选：如有额外 Python 依赖
└── utils/                 # 可选：如需拆分模块，可在此放置自定义工具文件
    ├── my_parser.py       # 框架不会覆盖 utils/ 下的选手自定义文件
    └── ...
```

### 9.2 注意事项

-   **禁止包含 `agent_base.py`** —— 该文件由框架提供，提交的版本不会被使用。
-   **禁止包含 API Key** —— 任何硬编码的密钥（`api_key = "..."`）会导致安全问题。
-   **`utils/` 目录可自由使用** —— 框架自带的 `utils/__init__.py`、`utils/image_utils.py`、`utils/visualize.py` 由主办方提供，你额外放置的文件不会被覆盖。
-   **不需要包含 `tasks/`、`test_runner.py` 等框架文件**，这些由主办方统一提供。
-   压缩包大小建议 < 10MB。

### 9.3 依赖管理

如果你的 `agent.py` 使用了框架 `requirements.txt` 以外的 Python 包，请在压缩包中附带 `requirements.txt`，仅列出额外依赖即可（框架已有的包无需重复列出）。

示例 `requirements.txt`：

```text
# 框架已包含 openai, pillow, matplotlib, numpy 等，无需重复
# 仅列出你的额外依赖
langchain>=0.3.0
```

## 十、常见问题

**Q: 中文输入怎么办？**

框架内置了 ADB Keyboard 中文输入支持。只需正常输出 TYPE 动作，传入中文文本即可。
手机上需预先安装 ADB Keyboard 输入法：

```bash
# 下载 APK: https://github.com/senzhk/ADBKeyBoard
adb install ADBKeyboard.apk

# 在手机上启用：设置 -> 语言和输入法 -> 虚拟键盘 -> 开启 "ADB Keyboard"

# 验证 (Windows PowerShell)
adb shell ime list -s | Select-String adbkeyboard
```

**Q: VLM_API_KEY 怎么获取？**

自测阶段使用你自己的 API Key。正式评测时主办方会使用统一的 Key，你的代码中不应包含任何 API Key。

**Q: 每步之间需要等待吗？**

框架在每步动作执行后自动等待 1 秒让界面稳定。如果你的任务需要更长的等待（如网络加载），可以使用 WAIT 动作。

**Q: 最大步数是多少？**

每个用例的最大步数在 `tasks.json` 中单独配置，根据题目难度最大步数在 30、50、80 不等。超过对应步数上限仍未返回 COMPLETE 的用例将以 `max_steps_reached` 结束。

**Q: 如果 Agent 出错了？**

框架会记录错误信息并终止当前用例，不影响后续用例执行。评委可以看到错误记录并在评分时酌情处理。

**Q: 有哪些 App 包名是支持的？**

支持的应用映射表见 `device_controller.py` 中的 `APP_PACKAGE_MAP`，覆盖了京东、淘宝、抖音、美团、拼多多、哔哩哔哩、百度地图、喜马拉雅、腾讯视频、QQ音乐等常见应用。使用 OPEN 动作时传入中文应用名即可。
